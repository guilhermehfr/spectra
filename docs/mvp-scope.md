# Spectra MVP Scope

## Objetivo

Entregar em 14 dias uma versão funcional do Spectra capaz de ser testada por uma clínica TEA real.

O MVP deve resolver os problemas centrais de organização operacional:

- agenda básica
- registro de sessões
- histórico centralizado de pacientes
- acompanhamento pela coordenação
- visibilidade simples para familiares

---

# Usuários do MVP

## Admin / Gestor

Responsável pela operação da clínica.

## Terapeuta

Profissional que realiza sessões e registra evoluções.

## Família

Responsável que acompanha atualizações do paciente.

---

# Funcionalidades que entram no MVP

## 1. Autenticação

- Login com e-mail e senha
- Perfis de acesso básicos

## 2. Dashboard

- Sessões do dia
- Pacientes ativos
- Evoluções pendentes

## 3. Pacientes

- Cadastro de paciente
- Dados básicos
- Responsáveis
- Histórico

## 4. Agenda

- Criar sessão
- Reagendar sessão
- Cancelar sessão
- Visualização semanal

## 5. Evolução terapêutica

Após sessão:

- objetivo
- atividades realizadas
- comportamento observado
- progresso
- próximos passos

## 6. Portal Família

- Visualizar últimas evoluções liberadas

---

# Funcionalidades que NÃO entram agora

- Financeiro
- Convênios
- Chat interno
- Notificações avançadas
- Relatórios PDF avançados
- IA
- Mobile app
- Permissões complexas
- Multiunidades

---

# Meta de validação

Ao menos 1 clínica testando o fluxo principal:

Agendar sessão → realizar atendimento → registrar evolução → coordenação acompanhar.

---

# Métricas de sucesso

- Tempo de registro da sessão < 2 min
- Sistema navegável sem suporte técnico
- Feedback positivo de usuário real
- Interesse em continuar teste

---

# Definição de pronto

O MVP estará pronto quando:

- login funcionar
- CRUD pacientes funcionar
- agenda funcionar
- registro de evolução funcionar
- portal família funcionar
- deploy online disponível
